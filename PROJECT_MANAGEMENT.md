# 项目管理规范

本文档用于规范本项目的代码管理、分支开发、提交记录与合并流程。所有开发人员应遵循以下规则，避免直接修改主分支并降低合并冲突风险。

## 1. 基本原则

1. 使用 GitHub Desktop 管理项目代码。
2. 禁止直接在 `Main` 分支上开发。
3. 每项开发任务都应从最新的 `Main` 分支创建独立的个人功能分支。
4. 开发过程中应及时进行本地保存，并养成小步提交的习惯。
5. 合并到 `Main` 分支前必须完成自检、同步和冲突处理。

## 2. 开发前准备

开始开发前，必须先同步最新的 `Main` 分支代码，确保本地基线是最新版本。

操作要求：

1. 打开 GitHub Desktop。
2. 切换到 `Main` 分支。
3. 拉取远端最新代码。
4. 确认本地 `Main` 分支已与远端保持同步。

## 3. 分支创建规范

每项开发任务都需要从 `Main` 分支创建新的个人功能分支，不允许多人长期共用同一个开发分支。

分支命名应清晰、规范，能够反映本次修改目的。

推荐命名格式：

```text
feature/xxx
fix/xxx
```

命名示例：

```text
feature/user-login
feature/data-visualization
fix/runtime-error
fix/ui-layout
```

分支类型说明：

| 类型 | 用途 |
| --- | --- |
| `feature/xxx` | 新功能开发 |
| `fix/xxx` | 问题修复 |
| `refactor/xxx` | 代码重构 |
| `docs/xxx` | 文档修改 |
| `test/xxx` | 测试相关修改 |

## 4. 开发与提交规范

开发过程中应及时进行本地保存，并采用小步提交方式记录修改。

提交要求：

1. 每次提交只包含一组相关修改。
2. 提交前检查本地改动，避免提交临时文件、缓存文件或无关文件。
3. 提交信息必须明确说明本次修改内容。
4. 不建议将大量不相关修改合并为一次提交。

提交信息示例：

```text
Add anomaly detection result export
Fix runtime path handling
Update environment setup document
Refactor main interface layout
```

## 5. 功能完成后的自检

完成开发后，必须先在个人分支上进行自检，至少包括以下内容：

- 核心功能可以正常运行。
- 已修改模块无明显报错。
- 不影响原有功能。
- 如项目有测试脚本，需完成对应测试。
- 如修改了界面、图表或输出文件，需确认结果符合预期。
- 如修改了依赖或环境配置，需同步更新相关说明文档。

建议自检记录：

```text
自检日期：
开发分支：
测试内容：
测试结果：
遗留问题：
```

## 6. 合并前同步流程

在准备合并到 `Main` 分支前，必须先检查远端 `Main` 是否有更新。

如远端 `Main` 有更新，必须执行以下流程：

1. 先拉取最新 `Main` 到本地。
2. 将最新 `Main` 合入个人分支，或基于最新 `Main` 重新同步个人分支。
3. 解决可能存在的冲突。
4. 冲突解决后，重新完整验证相关功能。
5. 确认无误后，再申请合并。

注意：冲突解决完成后不能只检查冲突文件，还需要重新验证本次开发涉及的完整功能。

## 7. 合并申请要求

申请合并前，应确认以下事项：

- 当前分支已同步最新 `Main`。
- 所有冲突已解决。
- 自检已完成。
- 无明显报错或异常输出。
- 无误提交临时文件、缓存文件或本地环境文件。
- 提交信息清晰，能够说明修改目的。

合并说明建议包含：

```text
本次修改内容：
影响范围：
自检结果：
是否涉及配置或依赖变更：
需要重点关注的问题：
```

## 8. 禁止事项

为保证项目稳定性，禁止以下操作：

1. 直接在 `Main` 分支开发功能。
2. 未同步最新 `Main` 就开始新任务。
3. 未自检就申请合并。
4. 将大量无关修改混入同一次提交或合并。
5. 提交缓存、临时输出、个人本地配置等无关文件。
6. 在不了解冲突含义的情况下随意覆盖他人代码。

## 9. 推荐工作流

```text
同步 Main
  -> 从 Main 创建个人分支
  -> 开发功能
  -> 小步提交
  -> 本地自检
  -> 合并前再次同步 Main
  -> 解决冲突
  -> 重新验证
  -> 申请合并
```

## 10. 检查清单

合并前可使用以下清单进行确认：

- [ ] 当前不在 `Main` 分支上直接开发。
- [ ] 开发分支从最新 `Main` 创建。
- [ ] 提交内容与本次任务相关。
- [ ] 提交信息清晰。
- [ ] 核心功能已验证。
- [ ] 修改模块无明显报错。
- [ ] 未影响原有功能。
- [ ] 如有测试脚本，已运行对应测试。
- [ ] 合并前已同步最新 `Main`。
- [ ] 如有冲突，已解决并重新验证。
- [ ] 未提交临时文件、缓存文件或本地环境文件。
