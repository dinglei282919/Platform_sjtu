# 过程仿真 DNN-MPC 模块使用说明

## 1. 模块概述

“化工控制过程”目录下的两个模块：

- `DNNTrain`：生成或导入过程数据，训练深度神经网络预测模型。
- `MPCsimulation`：调用训练后的模型，执行模型预测控制闭环仿真。

标准使用流程为：

```text
过程数据 → DNNTrain → DNN 预测模型 → MPCsimulation → 状态、控制量和性能指标
```

当前内置对象为四状态、单输入的非线性过程仿真模型：

$$
x=[x_1,x_2,x_3,x_4]^T,\qquad u\in\mathbb{R}
$$

`x1`～`x4` 和 `u` 均为抽象过程变量，不预设为某一具体装置中的温度、压力或流量。

默认流程中，内置非线性状态方程用于生成训练样本。训练完成后，`MPCsimulation` 使用同一 DNN 预测未来状态并推进闭环仿真状态。因此，当前结果属于模型在环过程仿真，用于评价学习模型与控制算法的闭环行为，不等同于独立机理模型或实际装置验证。

## 2. 算法原理与功能

### 2.1 DNN 过程预测

DNN 根据当前状态和当前操纵量预测下一采样时刻的状态：

$$
\hat{x}_{k+1}=f_{\mathrm{DNN}}(x_k,u_k)
$$

网络输入为 `[x1, x2, x3, x4, u]`，共 5 个变量；输出为下一时刻的 `[x1, x2, x3, x4]`，共 4 个变量。

DNN 学习的是过程的离散状态转移关系，即“施加某个操纵量后，过程状态将如何变化”。DNN 负责预测，不直接决定控制量。

### 2.2 MPC 滚动优化

MPC 在每个采样时刻利用 DNN 预测未来多个时刻的状态，并求解一组满足约束的候选控制量，使预测状态接近目标，同时限制控制作用。当前实现的目标函数为：

$$
J=\sum_{i=0}^{N_p-1}
\left[(\hat{x}_{k+i+1}-x_{\mathrm{ref}})^TQ(\hat{x}_{k+i+1}-x_{\mathrm{ref}})
+u_{k+i}^TRu_{k+i}\right]
+(\hat{x}_{k+N_p}-x_{\mathrm{ref}})^TP(\hat{x}_{k+N_p}-x_{\mathrm{ref}})
$$

其中：

- $N_p$：预测时域步数；
- $x_{\mathrm{ref}}$：目标状态；
- $Q$：状态跟踪误差权重；
- $R$：控制幅值权重；
- $P$：在预测末端额外施加的状态误差权重。

优化完成后只执行候选控制序列中的第一个控制量。进入下一采样时刻后，系统根据新状态重新预测和求解，这一过程称为滚动时域控制。

当前内置配置为：

$$
x_0=[0,0,\pi+0.1,0]^T,\qquad
x_{\mathrm{ref}}=[0,0,\pi,0]^T
$$

采样周期为 `0.02 s`。`π` 是当前内置算例为第三个状态设置的数值参考点，本文档不赋予其特定物理含义。结果图将第三个状态显示为偏差量 `x3 - π`，因此四条状态曲线均可按照“趋近于 0”进行观察。

## 3. 运行前配置

本文以软件平台已安装并可进入主界面为前提。运行两个模块前，应确认以下配置：

| 配置项 | 说明 | 默认或推荐值 |
|---|---|---|
| Python 包目录 | MATLAB 编译包所在目录 | `dnn_mpc/build_python` |
| Python 包名 | Python 调用的编译包名称 | `dnnmpcpkg` |
| `MCR_ROOT` | MATLAB Runtime 安装根目录 | 与编译版本一致的 MATLAB Runtime R2024b |
| 输出目录 | 模型、图像和汇总文件保存位置 | `dnn_mpc/output` |

若模块无法初始化，应优先检查 MATLAB Runtime 版本、`MCR_ROOT` 路径、包目录和包名。

## 4. DNNTrain 模块

### 4.1 界面参数

| 参数 | 含义 | 界面默认值 | 调整影响 |
|---|---|---:|---|
| 训练样本数 | 用于学习状态转移关系的数据条数 | 1000 | 增加样本可扩大数据覆盖范围，但会增加训练时间 |
| 训练轮数 | 网络参数更新的最大轮数 | 50 | 过少可能欠拟合；过多会增加耗时，并可能扩大过拟合风险 |
| 隐藏层结构 | 各隐藏层神经元数量 | `64,64` | 更大网络可能提高拟合能力，也会增加计算量和过拟合风险 |
| 外部数据文件 | 可选的 `.mat` 训练数据 | 空 | 为空时自动生成数据；选择文件时使用外部数据 |

隐藏层结构使用逗号分隔。例如，`64,64` 表示两个隐藏层，每层 64 个神经元；`128,64,32` 表示三个隐藏层。

### 4.2 外部数据格式

外部 `.mat` 文件应包含：

- `X_data`：尺寸为 `5×N` 或 `N×5`，每条数据为 `[当前四维状态, 当前操纵量]`；
- `Y_data`：尺寸为 `4×N` 或 `N×4`，每条数据为下一采样时刻的四维状态。

`X_data` 与 `Y_data` 的样本数必须相同，且至少包含 10 条样本。模块会依据数据集自身的均值和标准差进行标准化。

直接配合当前 `MPCsimulation` 使用时，外部数据还必须满足以下条件：

- 一步状态转移的时间间隔为 `0.02 s`；
- 状态顺序、状态尺度和操纵量定义与内置控制配置一致；
- 目标状态和 `-10 ≤ u ≤ 10` 的约束在相同坐标定义下有效。

仅满足“4 个状态、1 个输入”的维度要求并不足以保证模型兼容。更换过程对象时，应同步修改初始状态、目标状态、权重和约束，并重新构建程序包。

### 4.3 操作步骤

1. 在“化工控制过程”中进入 `DNNTrain`。
2. 检查包目录、包名、`MCR_ROOT` 和输出目录。
3. 设置样本数、训练轮数和隐藏层结构。
4. 如使用外部数据，选择符合要求的 `.mat` 文件；否则保持外部数据路径为空。
5. 启动训练并等待进度完成。
6. 确认输出目录中生成 `process_control_nn_model.mat`。

### 4.4 训练结果分析

| 结果 | 实际含义 | 较好的表现 |
|---|---|---|
| `training_performance.png` | 标准化输出空间中的训练集、验证集和测试集 MSE | 三类误差总体下降并趋于稳定，验证和测试误差与训练误差差距较小 |
| `prediction_error.png` | 每次训练更新后，在全部数据上计算的原始尺度综合误差 | 误差总体下降并趋于稳定，训练后段无持续增大或反复剧烈波动 |

`prediction_error.png` 中每个点的计算方式为：

$$
\frac{1}{N}\sum_{j=1}^{N}\lVert\hat{y}_j-y_j\rVert_2^2
$$

`training_summary.json` 中的 `mse_train` 为上述四维状态平方误差和的最终均值，并非独立测试集误差。由于两张图分别采用标准化尺度和原始状态尺度，其数值不能直接比较。

常见判断：

- 训练、验证误差均较大：可能存在欠拟合或数据覆盖不足。
- 训练误差较小，验证或测试误差明显较大：可能存在过拟合。
- 某些状态误差长期较大：应检查状态尺度、样本覆盖范围和数据质量。

## 5. MPCsimulation 模块

### 5.1 界面参数

| 参数 | 含义 | 界面默认值 | 调整影响 |
|---|---|---:|---|
| DNN 模型文件 | `DNNTrain` 生成的预测模型 | `process_control_nn_model.mat` | 必须与当前状态、输入和采样周期定义一致 |
| 仿真时间 | 闭环过程的总运行时间 | `1 s` | 时间越长，越容易观察后段趋势，计算量也越大 |
| 预测时域 | 每次优化向前预测的采样步数 | 5 | 当前对应 `0.1 s`；增大后考虑未来更充分，但优化耗时增加 |

### 5.2 内部控制参数

| 参数 | 当前设置 | 含义 |
|---|---:|---|
| 状态权重 `Q` | `diag(300, 1, 300, 1)` | 在当前变量尺度下，对第 1、3 个状态偏差给予更高惩罚 |
| 控制权重 `R` | `0.01` | 惩罚控制幅值；增大时通常会减小控制幅值，但不直接约束控制变化率 |
| 终端权重 `P` | `P = Q` | 对预测时域末端状态额外施加误差惩罚 |
| 操纵量范围 | `-10 ≤ u ≤ 10` | 优化器必须满足的输入幅值约束 |

当前界面不提供 `Q`、`R`、`P` 和输入范围的修改入口。如需调整，应修改 MATLAB 源码并重新构建程序包。权重变化不保证性能必然改善，修改后必须在相同条件下重新验证跟踪误差、输入饱和和计算开销。

### 5.3 操作步骤

1. 确认 `DNNTrain` 已成功生成预测模型。
2. 在“化工控制过程”中进入 `MPCsimulation`。
3. 检查运行环境和输出目录。
4. 选择与当前过程定义一致的 `process_control_nn_model.mat`。
5. 设置仿真时间和预测时域。
6. 启动仿真并等待计算完成。
7. 查看状态轨迹、控制输入、跟踪误差和代价曲线。

### 5.4 仿真结果与指标

| 结果图 | 含义 | 分析方法 |
|---|---|---|
| `process_control_trajectory.png` | 四个状态随时间的变化 | `x1`、`x2`、`x3-π`、`x4` 应逐步接近 0，且不发散或持续大幅振荡 |
| `control_input.png` | MPC 求得的操纵量 `u` | 应位于 `[-10,10]` 内；不宜长期饱和或持续剧烈变化 |
| `tracking_error.png` | $e_k=\lVert x_k-x_{\mathrm{ref}}\rVert_2$ | 初始过渡后应总体下降，并在仿真后段保持较低水平 |
| `cost_curve.png` | 每个采样时刻优化器返回的预测时域目标值 | 总体下降或进入较低区间表示预测误差和控制作用的综合代价有所改善 |

代价曲线不要求严格单调下降，因为每个采样时刻均基于新状态重新预测和求解。局部波动不能单独判定为控制失效。

主要汇总指标为：

- `tracking_mse`：$\frac{1}{N}\sum_{k=1}^{N}e_k^2$，即四维状态误差平方和的时间平均值。
- `cumulative_cost`：各采样时刻预测时域代价之和。相邻预测时域互相重叠，因此它不是实际过程能耗，只适合同一模型、仿真时间和预测时域下的相对比较。

## 6. 控制效果判定

控制效果应同时考察计算有效性、状态跟踪、输入约束和优化代价，不能仅凭单条曲线判断。

| 检查项 | 表明控制效果较好的现象 | 需要排查的现象 |
|---|---|---|
| 运行有效性 | 仿真正常结束，无 `NaN`、`Inf`，`exitflag_log` 中大部分求解状态为正 | 求解频繁失败、结果文件不完整或出现非有限数值 |
| 状态跟踪 | 仿真后段明显比初始阶段更接近目标，且所有状态保持有界 | 状态持续远离目标、发散或保持大幅振荡 |
| 跟踪误差 | 后 20% 时间段的平均误差低于前 20% 时间段 | 误差持续上升或后段没有改善 |
| 控制输入 | 始终满足幅值约束，过渡后无长期饱和 | 长时间处于上、下限或持续剧烈变化 |
| 优化代价 | 总体下降或在较低范围内稳定 | 与跟踪误差同步持续增大 |

可使用相对误差比辅助判断。令 $\bar{e}_{\mathrm{early}}$ 和 $\bar{e}_{\mathrm{late}}$ 分别表示仿真前 20% 和后 20% 时间段内 $e_k$ 的平均值，则：

$$
\eta=\frac{\bar{e}_{\mathrm{late}}}{\bar{e}_{\mathrm{early}}}
$$

- `η < 1` 表示误差相对初始阶段有所改善，但不能单独证明控制效果良好；
- `η` 越小，通常表示误差衰减越充分；
- `η ≥ 1` 时，应结合状态轨迹、输入饱和和 DNN 预测误差进行排查；
- 当分母为 0 或接近 0 时，不使用该比值，直接比较误差绝对值和状态轨迹。

不同状态可能具有不同量纲，因此不宜脱离变量尺度设置统一的绝对合格阈值。

当仿真有效完成、状态趋近目标、误差明显衰减、输入满足约束且无长期饱和、代价总体改善时，可以认为当前配置在模型在环条件下取得了较好的闭环控制效果。该结论不单独证明控制器在模型偏差、外部扰动或实际装置条件下的鲁棒性。

## 8. 使用注意事项与常见问题

1. 应先运行 `DNNTrain`，再运行 `MPCsimulation`；预测模型文件是两个模块之间的接口。
2. 每次调整参数时建议只改变一项，并在相同模型、初始状态和仿真时间下比较结果。
3. DNN 在训练数据覆盖范围外的预测可靠性可能下降，控制效果不佳时应先检查预测误差和数据覆盖范围。
4. 模块初始化失败时，检查 MATLAB Runtime R2024b、`MCR_ROOT`、Python 包目录和包名。
5. 外部数据无法读取时，检查变量名、数据维度、样本数和采样周期。
6. MPC 无法找到模型时，重新选择 `DNNTrain` 输出的 `process_control_nn_model.mat`。
7. 状态不收敛或控制量长期饱和时，依次检查 DNN 精度、模型兼容性、目标可达性和预测时域；内部权重或约束调整需要修改源码并重新构建。
8. 用于新的过程对象前，应重新定义状态、输入、初始状态、目标状态、采样周期、约束和评价指标，并采用对应数据完成训练和独立模型验证。
